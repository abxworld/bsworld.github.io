Git常用命令

2020-10-19 14:42
git
约 666 字 预计阅读 2 分钟 次阅读

文章目录

git不能commit空的文件夹，也就是说文件夹下必须要有文件才能commit

1、提交本地文件

1

git commit -a -m ""

2、查看本地已提交但未push到远程的文件信息

  • git status

    只能查看未push到远程的数量

  • git cherry -v

    可以查看到已提交但未提交到远程的commit 号，以及commit 备注信息，但无法看详细信息

  • git log branch_name origin/branch_name

可以查看到已提交但未提交到远程的详细信息

3、push本地分支到远程

1

git push origin branch_name

4、新建分支

1
2
3
4
5

git checkout -b branch_name

git push origin branch_name

git push --set-upstream origin branch_name

5、合并分支

1

git merge branch_name

6、查看本地文件变化

  • git status

    可以查看新增文件，以及修改的文件名称，但无法看见详细信息

  • git diff

    可以查看本地新增文件以及修改文件的改动的详细信息

7、丢弃本地文件的改动

  • git checkout .

    丢弃当前目录及子目录下的所有改动

  • git checkout file_path/file_name

    丢弃指定文件的改动

8、重新命名分支

假设分支名称为oldName想要修改为 newName

  1. 本地分支重命名(还没有推送到远程)

    1

    git branch -m oldName newName

  2. 远程分支重命名 (已经推送远程-假设本地分支和远程对应分支名称相同)

    重命名远程分支对应的本地分支

    1

    git branch -m oldName newName

  3. 删除远程分支

    1

    git push --delete origin oldName

  4. 上传新命名的本地分支

    1

    git push origin newName

  5. 把修改后的本地分支与远程分支关联

    1

    git branch --set-upstream-to origin/newName

  6. 修改最近一次的提交说明

    1

    git commit --amend -m "提交说明"

9、git分支删除

  1. 删除本地分支

    1

    git branch -D branch-name

  2. 删除远程分支

    以下两条命令都要执行才可以保证彻底删除:

    1
2
3

    git branch -r -d origin/branch-name
   
git push origin :branch-name               （注意:origin 后有空格）

10、新建

  1. 初始化本地仓库

    1

    git init

  2. 本地仓库与远程仓库关联

    1

    git remote add http://.....git

    如果报错:

    error: failed to push some refs to

    需要先执行:

    git pull origin

    如果报错:

    git refusing to merge unrelated histories

    执行:

    git pull origin branchname –allow-unrelated-histories

  3. 将本地代码推送到远程

    1

    git push  origin master
